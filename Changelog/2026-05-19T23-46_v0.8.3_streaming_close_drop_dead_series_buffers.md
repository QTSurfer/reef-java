# 0.8.3 — streaming close() + drop dead series buffers

## What
Two memory-related changes in `LastraWriter` so writers that produce thousands
of row-groups via repeated `writeSeries(...)` calls (the streaming pattern
introduced by `lastra-convert` 0.14.0) fit in modest heap budgets.

### 1. Drop dead series buffer retention (in `writeSeries`)
The block that appended one entry to `seriesLongBuffers / seriesDoubleBuffers
/ seriesBinaryBuffers` per call has been removed. Those lists were never
read by `close()` for series output — the comment on the prior code admitted
they were "not used when rowGroups > 0", and tracing showed they were not
used at all (the only readers of these fields were the writers themselves).
Holding the per-call buffer references kept the entire input dataset live in
heap for the duration of the writer, doubling residency for no benefit.

The fields and their declarations are also removed.

Events still buffer (`event*Buffers`) — `close()` reads from those for the
events payload.

### 2. Stream the body in `close()` instead of buffering it
The previous `close()` accumulated the entire serialized file (header +
column descriptors + every row-group's compressed bytes + footer) into a
`ByteArrayOutputStream`, then wrote it to `out` in one shot at the end.
This was the second OOM site for streaming callers: 2832+ row-groups × 11
columns of compressed bytes plus `BAOS.ensureCapacity`'s doubling
amplification could push residency past 500 MB on an 11.6M-row input.

Replaced with:

- `BufferedOutputStream(out, 64 KiB)` wraps the consumer-supplied stream.
- A small `CountingOutputStream` wraps that, exposing an absolute `position`
  long so the writer knows current byte offsets without seeking back.
- Helpers `writeIntLE / writeLongLE / writeShortLE / writeColumnDescriptors`
  now take `OutputStream` (used to take `ByteArrayOutputStream`); they
  declare `throws IOException`.
- All previous `body.size() - dataStart` arithmetic becomes
  `(int)(counter.position - dataStart)`.
- `rgOffsets` switched from `List<Integer>` to `int[]` (one int per RG,
  trivially small in heap).

## Validation
- 26/26 existing unit tests pass (`LastraRoundtripTest`, `LastraReferenceFixtureTest`,
  `GorillaCodecTest`, `PongoCodecTest`).
- End-to-end test from `lastra-convert` 0.14.0:
  - Input: 286 MB / 11,609,190-row Parquet file.
  - `java -Xmx512m -jar lastra-convert-0.14.0.jar in.parquet out.lastra --smart` → success in 25 s.
  - Output: 158 MB Lastra (1.8× compression vs Parquet).
  - Roundtrip via `LastraToParquetConverter` returns the same row count
    (11,609,190 == 11,609,190).

## Compat
- File format on disk is unchanged.
- Public `LastraWriter` API unchanged (callers don't reference the private
  helpers or the removed fields).
- Events path is untouched.
